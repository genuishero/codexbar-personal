## 🎯 产品方向补充建议

### 方向 1：**安全保障层（市场信任前提）**

#### P0 - 必须有，否则无法商业化
| 功能 | 说明 | 技术方案 |
|------|------|----------|
| **macOS Keychain 存储** | 替换明文 JSON 存储 | `SecItemAdd` / `SecItemCopyMatching` |
| **主密码解锁** | 启动时要求密码，内存 clearing | PBKDF2 + AESGCM 加密 config |
| **剪贴板自动清除** | 复制后 30 秒自动清空 | `NSPasteboard` 监听 + Timer |
| **配置完整性校验** | 防篡改 HMAC-SHA256 | 保存时加 MAC，加载时验证 |

#### P1 - 信任增强
| 功能 | 说明 |
|------|------|
| **安全状态指示器** | 设置页显示"🔒 加密存储"、"🛡️ 剪贴板保护"等 |
| **安全日志** | 记录 Token 清除、配置修改等操作 |
| **App Sandbox** | 启用文件访问限制，避免越权读取 |

---

### 方向 2：**用户体验层（差异化竞争力）**

#### P0 - 基础体验
| 功能 | 说明 |
|------|------|
| **首次使用引导** | 3 步引导卡片（添加 Provider → 添加账号 → 开始使用） |
| **快捷键支持** | `Cmd+Shift+1~5` 快速切换账号 |
| **错误提示友好化** | 技术错误转用户语言（如 "Key 格式有误" → "这个 API key 看起来不对劲"） |

#### P1 - 进阶体验
| 功能 | 说明 |
|------|------|
| **用量阈值预警** | Token 用到 80% 时弹窗提醒 |
| **账号分组标签** | 按项目/客户分组账号（如 "项目A"、"客户B"） |
| **自动 Token 刷新** | OAuth Token 过期前自动刷新 |

#### P2 - 精细体验
| 功能 | 说明 |
|------|------|
| **使用报表导出** | 支持导出 CSV/PDF 用量统计（报销用） |
| **深色模式优化** | 自动跟随系统主题，提供专属深色主题 |
| **动画反馈增强** | 切换账号时的视觉反馈（绿色闪烁 + Toast） |

## 🎯 决策建议

### 如果你是 solo 开发者
**推荐选项 A**：先做 8 天的 MVP 把安全问题解决，快速上线建立信任。Pro 版本可以交给社区贡献或等有用户基础后再做。

### 如果你有团队/资金
**推荐选项 B**：2周安全 + 2周 Pro 功能，一次性发布完整产品。宣传时强调"专业级 AI 账号管理平台"。

### 如果你 Focus 做市场
**推荐选项 C**：直接启动 Team/Enterprise 版本
- 跳过个人市场，直接对接企业客户
- 定价更高（$15+/人），客单价提升
- 需要更多功能（SSO、审计日志、集中管理）

## 📌 主要风险点

| 风险 | 影响 | 应对策略 |
|------|------|----------|
| **安全问题未解决** | 用户流失 + 负面口碑 | MVP 优先做 Keychain + 主密码 |
| **Apple App Store 审核** | 上架延迟 | 用 DMG 分发，避开 Store 审核 |
| **OpenAI API 政策变化** | 功能失效 | 保持代码模块化，快速响应变化 |
| **竞品快速跟进** | 市场份额丢失 | 快速迭代，建立品牌认知 |
